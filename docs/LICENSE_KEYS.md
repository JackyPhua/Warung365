# 授权码（License Key）说明

本文说明 Warung365 POS 离线授权码的格式、有效期与生成方式。实现代码见 `src/services/LicenseService.js`。

## 有效期（月数）

激活后从 **当日** 起算，按 **自然月** 递增（`Date.setMonth`），不是固定 30 天。

| 期限   | 密钥第二段标记 |
|--------|----------------|
| 1 个月 | `01MT`         |
| 3 个月 | `03MT`         |
| 6 个月 | `06MT`         |
| 12 个月 | `12MT`        |

## 密钥格式

### 新格式（推荐）

```
POS-{期限标记}-XXXX-XXXX
```

- 第一段固定为 `POS`。
- 第二段为上表中的 `01MT` / `03MT` / `06MT` / `12MT`。
- 第三、四段各 4 位，字符集为 `A–Z` 与 `0–9`。
- **校验**：仅对 **最后 8 个字符**（第三段 + 第四段）做 ASCII 码加总，**和能被 7 整除** 则通过。

示例（勿直接复制使用，请用脚本生成）：

```text
POS-12MT-XXXX-XXXX
```

### 旧格式（兼容）

```
POS-XXXX-XXXX-XXXX
```

三段各 4 位随机字符；**校验**为 **12 位** 连续字符的 ASCII 加总能被 7 整除。验证通过后一律视为 **1 个月**。

### 演示用固定密钥

以下写死在应用中，可用于测试：

- `POS-DEMO-TEST-KEY1`
- `POS-ALPHA-2024-MAIN`
- `POS-BETA-2024-MAIN`

均按 **1 个月** 从激活日起算。

## 如何生成新密钥（给门店）

在项目根目录执行（参数只能是 `1`、`3`、`6`、`12`）：

```bash
npm run license:gen -- 12
```

或：

```bash
node scripts/generate-license-key.mjs 6
```

终端会打印一条合法密钥；可重复执行以获得多条。

## 在 App 中使用

打开 **设置 → 授权**，输入密钥后点 **激活**。成功后会显示约「授权有效期：N 个月（自激活日起）」之类提示。

## 技术备注

- 授权状态与本机存储键 `pos_state_v3` 一并持久化。
- 无线上服务器校验；适用离线单机或局域网场景。
